## Working conventions (how Plans 1–2 were built — keep consistent)
- **Module path:** `github.com/diagridio/dev-dashboard`. **Go toolchain:** 1.26.x (go.mod). **Node:** 20 (build-time only).
- **Plan authoring:** superpowers `writing-plans` — bite-sized TDD tasks with full code, Global Constraints block copied from the spec, self-review.
- **Execution:** superpowers `subagent-driven-development` — one git worktree per plan off `main` (`git worktree add .claude/worktrees/<name> -b feat/<name>`), fresh implementer + reviewer subagent per task, fix subagents for Critical/Important, whole-branch review at the end, then `finishing-a-development-branch` (merge to `main` + push). Progress tracked in a `.superpowers/sdd/progress.md` ledger inside the worktree (gitignored).
- **Tests:** Go `testing` + `testify`, build tag `//go:build unit`; run `go test -tags unit ./...`. Frontend Vitest + Testing Library + MSW; `data-cy` selectors. **Vitest does NOT typecheck — also run `cd web && npm run build` (tsc+vite)** since CI's web job does.
- **CI gates:** `gofmt -l` (must be empty), `go vet -tags unit ./...`, `gotestsum -race`, web build+test. Run `gofmt -w` before committing Go.
- **Commit hygiene:** never `git add web/dist/` (only `web/dist/index.html` is a tracked static placeholder; built assets are gitignored and regenerated by `make build`). Commit messages end with the Co-Authored-By trailer.
- **State-store testing (Plan 3):** reuse cloudgrid conventions — `miniredis` (Redis), real temp-file SQLite, `testcontainers-go`/`go-sqlmock` (Postgres), behind `//go:build integration`.
