# React 18 → 19 Upgrade Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Upgrade the `web/` SPA from React 18.3 to React 19, verify the build/tests/peers still pass, and adopt the two clean React 19 idioms available in the codebase.

**Architecture:** Bump `react`/`react-dom` and their `@types` to v19 in `web/package.json`, regenerate the lockfile, and confirm the existing gates (`tsc -b`, `vite build`, `vitest`) stay green. Then make two small modernizations in `src/lib/refresh.tsx`: context-as-provider and the `use()` hook.

**Tech Stack:** React 19, Vite 8, TypeScript 5.5, Vitest 4, @testing-library/react 16, @tanstack/react-query 5, react-router-dom 6.

## Global Constraints

- All commands run from the `web/` directory unless noted otherwise.
- Peer dependencies stay at current major versions: `@tanstack/react-query@^5`, `react-router-dom@^6`, `@testing-library/react@^16`, `@vitejs/plugin-react@^6`. Do **not** bump them.
- Target React version: `^19` for `react`, `react-dom`, `@types/react`, `@types/react-dom`.
- No new dependencies.
- Gates that must stay green: `npm run build` and `npm test`.

### Task 1: Bump React to v19 and verify gates

**Files:**
- Modify: `web/package.json` (dependencies + devDependencies)
- Modify: `web/package-lock.json` (regenerated by npm)

**Interfaces:**
- Consumes: nothing (first task).
- Produces: a working install on React 19 — Task 2 edits code that compiles against `@types/react@19`.

- [ ] **Step 1: Establish the current baseline is green**

Run (from `web/`):
```bash
npm run build && npm test
```
Expected: both succeed on React 18. If anything fails here, stop — that failure is pre-existing and out of scope for this upgrade.

- [ ] **Step 2: Edit `web/package.json` version ranges**

In `dependencies`, change:
```json
    "react": "^19",
    "react-dom": "^19",
```
In `devDependencies`, change:
```json
    "@types/react": "^19",
    "@types/react-dom": "^19",
```
Leave every other entry untouched.

- [ ] **Step 3: Install and regenerate the lockfile**

Run (from `web/`):
```bash
npm install
```
Expected: install completes. If npm reports peer-dependency errors (it should not — all peers support React 19), do **not** add `--legacy-peer-deps`; instead record the exact error and stop for review.

- [ ] **Step 4: Confirm the installed versions**

Run (from `web/`):
```bash
npm ls react react-dom
```
Expected: both resolve to a `19.x` version.

- [ ] **Step 5: Run the build (typecheck + bundle)**

Run (from `web/`):
```bash
npm run build
```
Expected: PASS. If `tsc -b` reports type errors from `@types/react@19` (most likely stricter `ReactNode`/`useRef`/event typings), fix each one minimally at its source — do not loosen types with `any`. Re-run until green.

- [ ] **Step 6: Run the tests**

Run (from `web/`):
```bash
npm test
```
Expected: all vitest suites PASS.

- [ ] **Step 7: Manual smoke test under React 19 StrictMode**

Run (from `web/`):
```bash
npm run dev
```
Open the served URL. Verify: pages render, routing works, and the live-refresh indicator behaves (toggling/pausing refresh). React 19 changes StrictMode double-invoke and effect timing, so this catches runtime regressions that types and unit tests miss. Stop the dev server when done.

- [ ] **Step 8: Commit**

Run (from repo root):
```bash
git add web/package.json web/package-lock.json
git commit -m "chore(web): upgrade react and react-dom to v19"
```

### Task 2: Adopt React 19 context idioms in refresh.tsx

**Files:**
- Modify: `web/src/lib/refresh.tsx`
- Modify: `web/src/components/LiveIndicator.test.tsx` (provider JSX in the `wrap` helper)

**Interfaces:**
- Consumes: React 19 from Task 1 — `<Context>` is usable directly as a provider, and `use(Context)` reads context.
- Produces: no API change. `RefreshContext`, `RefreshProvider`, `useRefreshInterval`, and `refetchMs` keep identical signatures; only internals change.

- [ ] **Step 1: Run the affected tests to confirm they pass before the change**

Run (from `web/`):
```bash
npm test -- src/components/LiveIndicator.test.tsx
```
Expected: PASS (this is the regression guard for the refactor).

- [ ] **Step 2: Switch the provider to context-as-provider in `refresh.tsx`**

In `web/src/lib/refresh.tsx`, replace the `RefreshProvider` return block:
```tsx
  return (
    <RefreshContext.Provider value={{ intervalMs, paused, setInterval, setPaused }}>
      {children}
    </RefreshContext.Provider>
  )
```
with:
```tsx
  return (
    <RefreshContext value={{ intervalMs, paused, setInterval, setPaused }}>
      {children}
    </RefreshContext>
  )
```

- [ ] **Step 3: Switch `useContext` to `use` in `refresh.tsx`**

Change the import line:
```tsx
import { createContext, useContext, useState, type ReactNode } from 'react'
```
to:
```tsx
import { createContext, use, useState, type ReactNode } from 'react'
```
Then in `useRefreshInterval`, change:
```tsx
  const ctx = useContext(RefreshContext)
```
to:
```tsx
  const ctx = use(RefreshContext)
```

- [ ] **Step 4: Update the provider JSX in the test**

In `web/src/components/LiveIndicator.test.tsx`, in the `wrap` helper, replace:
```tsx
    <RefreshContext.Provider
      value={{ intervalMs, paused, setInterval: () => {}, setPaused: () => {} }}
    >
      <LiveIndicator />
    </RefreshContext.Provider>,
```
with:
```tsx
    <RefreshContext
      value={{ intervalMs, paused, setInterval: () => {}, setPaused: () => {} }}
    >
      <LiveIndicator />
    </RefreshContext>,
```

- [ ] **Step 5: Run the affected tests**

Run (from `web/`):
```bash
npm test -- src/components/LiveIndicator.test.tsx
```
Expected: PASS.

- [ ] **Step 6: Run the full build and test suite**

Run (from `web/`):
```bash
npm run build && npm test
```
Expected: both PASS — confirms no typecheck or other-suite regression from the context changes.

- [ ] **Step 7: Commit**

Run (from repo root):
```bash
git add web/src/lib/refresh.tsx web/src/components/LiveIndicator.test.tsx
git commit -m "refactor(web): adopt React 19 context-as-provider and use() in refresh"
```
